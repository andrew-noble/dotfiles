Navigation Philosophy

Every layer of the stack uses the same spatial vocabulary — h goes left, l goes right — with a different modifier that gets lighter as you go deeper. Muscle memory transfers across contexts.

The Stack

nvim buffers

nvim panes
H / L  (capital, normal mode)

tmux

KeyAction
Alt+h / Alt+lcycle windows left / right
Prefix + h/j/k/lmove between panes
Ctrl+aprefix (alternate, alongside Ctrl+b)

Alt+h/l for windows is modifier-only — no prefix — so it's fast. Pane movement requires the prefix, which adds just enough friction to feel intentional.

nvim

KeyAction
Tab / Shift+Tabcycle buffers left / right
H / Lmove between vertical panes

Capital H/L repurpose seldom-used defaults (screen-top / screen-bottom jumps) for pane movement, keeping navigation on the home row with no modifier at all.

Synergy with the OS

Super+Tab sits at the top of the hierarchy — same left/right logic, heaviest modifier. The mental model is consistent: heavier modifier = higher layer of the stack. Super owns the OS, Alt owns tmux windows, bare keys own the terminal's interior.
